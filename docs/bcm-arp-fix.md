# BCM Dual NIC ARP Fix

When BCM has two NICs on the same VLAN and subnet, Linux's weak host model causes ARP replies for eth0 (172.16.103.201) to route out eth1, making eth0 unreachable via unicast. This breaks PXE provisioning since compute nodes cannot reach the TFTP/DHCP server after receiving a DHCP lease. The fix applies `arp_filter=1` and `arp_announce=2` sysctls to enforce strict per-interface ARP behavior, combined with source-based policy routing (ip rule + table 100) to ensure traffic originating from eth0's IP always egresses via eth0. After applying, both 172.16.103.201 (provisioning) and 172.16.103.202 (management) respond correctly to ARP and ICMP from hosts on the same VLAN. The fix is persisted via `/etc/sysctl.d/99-arp-fix.conf` and `post-up` commands in the eth0 interface config.
